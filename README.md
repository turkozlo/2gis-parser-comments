# 2GIS Parser для Сбербанка

Проект для парсинга отзывов о отделениях Сбербанка с сайта 2GIS с последующим анализом через LLM (Mistral AI) и генерацией расширенной отчетности.

## 🚀 Основные возможности

- **Парсинг 2GIS**: Сбор отзывов, рейтингов, дат и данных пользователей.
- **LLM Тегирование**: Автоматическая классификация отзывов по категориям (#Обслуживание, #Технические_проблемы и т.д.).
- **Анализ подтем**: Выявление конкретных проблем внутри каждого тега (например, "Очереди" внутри "Обслуживания").
- **Гео-аналитика**: Автоматическое определение ГОСБ (региона) по адресу отделения.
- **Расширенная отчетность**: Генерация иерархической хитмапы и анализа проблем в Excel.
- **Уведомления**: Отправка отчетов в Telegram и на Email.

## 📂 Структура проекта

```
2gis-parser-for-pred/
├── src/                          # Исходный код
│   ├── parsers/                  # Парсеры (основной: parse_reviews.py)
│   ├── processors/               # Обработка данных (LLM, отчеты)
│   │   ├── subtopic_classifier.py # Классификация подтем
│   │   ├── topic_modeler.py      # Выявление новых тем через LLM
│   │   └── report_generator.py   # Генерация консольных отчетов
│   ├── filters/                  # Фильтры данных
│   ├── email_notifier.py         # Отправка отчетов на почту
│   └── telegram_notifier.py      # Отправка отчетов в Telegram
├── analyze_subtopics.py          # Скрипт для выявления топ-5 подтем
├── backfill_subtopics.py         # Скрипт для заполнения подтем в старых данных
├── generate_heatmap.py           # Генерация Excel-отчета (Хитмапа + Анализ)
├── analyze_positive_service.py   # Анализ позитивных отзывов по теме сервиса
├── sberbank_DVB_VSP.csv         # Входной файл с отделениями
├── sberbank_all_reviews.csv     # Основная база отзывов
└── config.json                   # Конфигурация (API ключи, Email, Telegram)
```

## 📊 Выходные данные (sberbank_all_reviews.csv)

- **GOSB** - Региональное подразделение (определяется автоматически).
- **Tags** - Основные категории жалоб (Mistral AI).
- **Sub_tag** - Конкретная подтема проблемы (Mistral AI).
- **Review Text, Rating, Date, Username, User ID** - Стандартные данные отзыва.

## 🛠️ Использование

### 1. Парсинг и базовый анализ
Запуск основного процесса сбора данных и тегирования:
```bash
python src/parsers/parse_reviews.py
```

### 2. Анализ подтем (Sub-topics)
Если нужно обновить список подтем на основе накопленных данных:
1. Выявить новые темы: `python analyze_subtopics.py`
2. Обновить `subtopics.json`: `python extract_subtopics.py`
3. Заполнить подтемы в базе: `python backfill_subtopics.py`

### 3. Генерация Excel-отчета
Создание хитмапы и автоматического анализа проблем за 2025 год:
```bash
python generate_heatmap.py
```
Результат: `heatmap_report.xlsx` (Листы: "Heatmap", "Анализ проблем").

### 4. Анализ позитива
Проверка позитивных отзывов на упоминание качественного сервиса:
```bash
python analyze_positive_service.py
```
